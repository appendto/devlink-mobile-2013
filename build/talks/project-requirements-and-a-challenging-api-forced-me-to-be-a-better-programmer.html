Project requirements and a challenging API forced me to be a better programmer

Friday
9:30 AM – 10:45 AM
Meeting Room 12

Every so often, you encounter a technical challenge with enough complexity that it makes you think... "Surely there must be a better way to do this." Sometimes, that's just the inner programmer talking, but occasionally, the problem is serious enough that you need to step up your game. This session profiles a series of challenges that lead me on one such journey. As the story opens, we find ourselves in kingdom of JavaScript. We begin with the basics for survival... object literals, helper functions, variable scoping, OOP... but come prepared adventurer; things are going to take a turn for the complex. AJAX Rate Limiters, Dynamic Object Creation, jQuery Promises, Templating, KendoUI controls & the Google Analytics API all make their way into this interactive demo. While the session is not specifically "about" any one of these technologies, this will be a Code Heavy presentation and as such, it is STRONGLY recommended that you have experience in one or more of the these technologies, or at least have a strong working knowledge of a C based language. I will make every attempt to explain each of the concepts in play here, but I don’t intend to focus on the syntax of any given library.

AJAX API Google Analytics JavaScript KendoUI Problem Solving Advanced JQuery

James Carpenter

Though he is a geek at heart, James has a very different perspective on how technology should work; it exists for the single purpose of making our lives easier, yet it often does just the opposite. Technology should not be a burden to use, it should not be slow and those who create it should not make excuses for it. Technology can do just about anything; the challenge will always be to make it easy to use, easy to understand and easy to love. As the Director of Technology at MoJo Active, it is his mission to overcome this challenge by developing innovative solutions for each and every project.
