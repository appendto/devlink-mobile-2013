So You Want To Build For Developers?

Thursday
2:30 PM – 3:45 PM
Meeting Room 3

Every day we use developer tools and might even create a few. API companies are one of the "exciting new things" in The Valley and libraries are being built and open sourced like crazy. However, creating a developer tool can be hard. It requires a lot of thought be put into API design, documentation, and more. After building one you must put a great deal of time into maintaining it, making architectural decisions, and managing its community. This talk explores some best practices in creating and maintaining developer tools, discovered through the creation and maintenance of open source libraries and put into action at major API companies. It will discuss common pitfalls in developer tools and uncover methods to sustainably develop libraries and APIs.

API Design Community Development Practices Documentation Langauge Agnostic Soft Skills Intermediate

Nicholas Quinlan

Hi, I'm Nick. I'm a SendGrid Developer Evangelist. I spend my days with APIs and nights with hackers. I love helping developers and building cool stuff.
