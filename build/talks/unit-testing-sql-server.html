Unit Testing SQL Server

Wednesday
2:30 PM – 3:45 PM
Meeting Room 6

In the nearly 15 years I have been working with SQL Server, most of that as an independent consultant, I can count two hands the number of clients I have come across that had ANY database tests constructed. I can count on TWO FINGERS the number of them that had repeatable database tests. Even the most ardent Agile/Scrum/pick-your-new-and-shiny-development-mantra shops hardly ever do anything more than wave a magic wand at testing the database, because it can be REALLY HARD to do and do correctly. In this session we will cover examples of why repeatable database testing is CRITICALLY important and then examine a free and a low-cost commercial product to ease the pain and improve developer efficiency.

Database Development SQL Server TSQL Testing Intermediate

Kevin Boles

Kevin is a SQL Server expert, working exclusively with the product since v6.5. With over 20 years of database experience and almost 45,000 man hours of SQL Server engine experience, he holds many related certifications, is an MCT and was a SQL Server MVP from 2007 to 2012. He has been a very successful independent consultant for over 15 years. Kevin is also a very frequent speaker, having presented at countless User Groups and over 75 SQL Saturday events! His passion is the relational engine, especially designing, building, analyzing and tuning high-performance database applications.
