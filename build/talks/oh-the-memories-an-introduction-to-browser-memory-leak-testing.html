Oh the Memories! - An Introduction to Browser Memory Leak Testing

Wednesday
4:00 PM – 5:15 PM
Meeting Room 3

Conferences like devLink are a wonderful time filled with many lasting memories. Just like how our brains can be overwhelmed by all of the great experiences at a conference, our web applications can also suffer from trying to remember too many things. Browser memory leaks are an often overlooked, yet important, issue in modern web development that can be overcome with the right techniques. In this session, we will take an introductory look at the tools and approaches that you can use to identify and investigate browser memory leaks. We will look at common scenarios that cause this problem, as well as patterns to use in your code to avoid leaks. Additionally, we will explore the various options for debugging and profiling leaks across different browsers and platforms. You'll leave this session with some new memories that will help you start tackling leaks in your web apps today!

JavaScript Memory Profiling Intermediate

Ryan Niemeyer

Ryan Niemeyer has over 15 years of experience in the software industry working with primarily web-based technologies. He loves JavaScript, participating in the open source community, and trying to learn new things everyday. He is always willing to lend a helping hand to anyone that needs it. Since, 2010 Ryan has been involved with the Knockout.js project and writes about his experiences with it on his blog (http://knockmeout.net). He works as an architect for appendTo.
