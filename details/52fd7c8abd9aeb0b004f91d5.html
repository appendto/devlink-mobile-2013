Automating database migrations

  • advanced automated deployment database migrations psake sql server
Are you using a shared database for development or fighting with your ORMs way of doing migrations? Looking for a way to automate your database changes between development, test, and production? This session will discuss how you can setup automated database migrations with simple SQL scripts that will make your DBA happy. Learn how each developer on your team can have a local copy of the database and maintain the schema changes in source control. Finally, we will go over some of the more difficult parts of migrations such as schema changes that effect data and how to handle test data for local development. This session primarily targets ASP.Net / C# / Microsoft SQL Server development and will include some demos using the Psake build automation tool and the AliaSQL database change management tool.

Eric Coffman

Eric has been involved in Microsoft based development for 15 years. He spent many years working as a systems administrator and has a unique perspective as a developer working at the intersection of devops. His day job at West Virginia University involves working on a custom learning management system developed in ASP.Net MVC. He also has a few open source projects including AliaSQL and SimpleCRUD. Recently he has spent considerable effort working on, developing tools for, and writing about automated deployment. He also likes to build Lego.
